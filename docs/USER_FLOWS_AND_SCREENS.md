# User Flows and Screens

## Student Flow
1. Student logs in.
2. If the student has no effective halaqa group assignment for the week, the student sees an assignment-pending screen.
3. Student lands on today's check-in page.
4. Student sees today's date, cohort/group, and this week's assigned teacher when one has been assigned.
5. Student saves checklist items and may add an optional note.
6. System shows confirmation and timestamp.
7. Student can view their own weekly history.
8. Student can view weekly grades, rewards, weekly plans, partner recitation, and sanitized leaderboard standings.
9. Student leaderboard standings include only students in the same cohort for the selected week.

## Admin Flow
1. Admin logs in.
2. Admin sees students in masajid where they have active admin membership.
3. Admin can filter by date, student, or completion status.
4. Admin can click a student to view detailed history.
5. Admin can manually correct a check-in.
6. Admin can enter Saturday halaqa grades, view weekly plans, rewards, and incentive reports for scoped students.
7. Admin can export scoped leaderboard data as CSV.
