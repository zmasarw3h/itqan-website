# User Flows and Screens

## Student Flow
1. Student logs in.
2. Student lands on today's check-in page.
3. Student sees today's date.
4. Student clicks "I completed today's work."
5. Student may add an optional note.
6. Student submits.
7. System shows confirmation and timestamp.
8. Student can view their own weekly history.

## Student Screens
- Login
- Today Check-In
- My History

## Admin Flow
1. Admin logs in.
2. Admin sees all students in a weekly table.
3. Admin can filter by date, student, or completion status.
4. Admin can click a student to view detailed history.
5. Admin can manually correct a check-in.
6. Admin can export check-in data as CSV.

## Admin Screens
- Admin Dashboard
- Student Detail
- CSV Export
